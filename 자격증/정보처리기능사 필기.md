# 전자계산기 일반

## 컴퓨터의 주기억 소자 발전 과정

1세대 : 진공관  
2세대 : 트랜지스터(TR)  
3세대 : 집적회로(IC)  
4세대 : 고밀도 집적회로(LSI)  
5세대 : 초고밀도 집적회로(VLSI)

## 자료처리 방식의 발전 과정

1. 배치처리(일괄처리, Batch) : 처리할 데이터들을 한 번에 모아 처리하는 것
2. 실시간(Real Time) : 처리할 데이터가 들어올 때마다 바로바로 처리하는 것
3. 다중 프로그래밍(Multi programming) : CPU 1개로 다중 작업
4. 다중 프로세싱(Multi processing) : CPU 2개 이상으로 다중 작업
5. 시분할(Time sharing) : 한 컴퓨터의 자원을 여러 사용자에게 적절히 배분하여 공동으로 동시에 사용할 수 있게 처리하는 것
6. 분산처리(Network) : 지역적으로 분산시켜 공동으로 하나의 작업을 처리하는 것

## 컴퓨터의 역사

1. Eniac(에니악) : 최초의 전자계산기로 외장방식과 진공관이 핵심
2. Edsac(에드삭) : 최초의 프로그램 내장방식 컴퓨터
3. Univac I(유니박 I) : 최초의 상업용 컴퓨터
4. Edvac(에드박) : 최초의 2진법 컴퓨터이며, 폰 노이만이 설계에 참여

## 컴퓨터의 분류 방식

- 데이터에 따른 구분  
아날로그 : 연속적인 물리량, 미분, 속도가 빠름  
디지털 : 셀 수 있는 문자와 숫자, 논리회로  
하이브리드 : 아날로그와 디지털의 장점을 혼합한 형태
- 목적에 따른 구분  
전용(특수목적)과 범용(일반목적)의 컴퓨터
- 능력에 따른 구분  
슈퍼컴퓨터(기상, 우주탐사)  
대형(메인프레임)  
중형, 소형(기업, 은행)  
미니(워크스테이션)  
마이크로(개인용)

## 컴퓨터의 구성

### 하드웨어

- 중앙처리장치(CPU) : 제어장치, 연산장치(ALU), 주기억장치(RAM), 레지스터로 구성되어있음
- 제어장치는 명령어 해독과 명령 순서를 결정하며, 명령 레지스터(IR)와 프로그램 카운터(PC) 등의 레지스터를 내포함
- 연산장치는 산술 및 논리 연산을 수행하며, 누산기, 가산기, 보수기, 상태 레지스터 등의 레지스터를 내포함
- 그 외에 주변장치(입출력장치, 보조기억장치)가 존재

### 소프트웨어

- 시스템소프트웨어(OS) : 컴퓨터와 사용자간의 중재자 역할을 하며, 인터페이스를 표시함
- 제어프로그램은 감시, 작업관리, 데이터관리 등을 수행
- 처리프로그램은 언어번역, 서비스 등을 수행
- 대표적인 운영체제엔 윈도우, DOS, UNIX, LINUX, OS/2, MAC 등이 존재
- 응용소프트웨어는 일반적인 사용자가 사용하는 워드, 파워포인트, 포토샵 등을 의미

### 레지스터

- 레지스터는 CPU 내부에서 CPU의 속도 향상을 위해 존재하는 기억장치
- 명령 레지스터(IR) : 현재 수행중인 명령어를 보관
- 프로그램 카운터(PC) : 다음에 수행할 명령어의 번지를 보관
- 누산기(ACC) : 산술 및 논리 연산의 중간 결과를 일시적으로 기억
- 메모리 주소 레지스터(MAR) : 메모리 액세스 시 특정 워드의 주소가 MAR에 전송
- 메모리 버퍼 레지스터(MBR) : 레지스터와 외부 장치 사이에서 전송되는 데이터의 통로

## 논리회로

- AND, OR, NOT 등을 기본 논리회로, 부울 연산자라고 함
  - AND : 논리곱, 직렬. 둘 다 1이어야 1
  - OR : 논리합, 병렬. 둘 중 하나만 1이어도 1
  - XOR : 배타적논리합, 같을 때 0, 다를 때 1
  - NOT : 인버터, 1이면 0, 0이면 1
  - NAND : AND(논리곱)의 결과에 NOT
  - NOR : OR(논리합)의 결과에 OR
- 이항연산 : 입력이 두 개인 논리회로 (AND, OR, XOR, ADD, SUB, MUL, DIV)
- 단항연산 : 입력이 한 개인 논리회로 (MOVE, Rotate, Complement, NOT, Shift, Buffer)
- 산술연산 : 더하기, 빼기, 곱하기, 나누기, 나머지

## 불(BOOL) 대수

 - A + 1 = 1
 - A + 0 = A
 - A + A' = 1
 - A * A' = 0
 - A * A = A
 - A + A = A

## 드모르간 정리

- (A * B)' = A' + B'
- (A + B)' = A' * B'

## 조합 논리회로

- 반가산기 : XOR(S), AND(C) 두 개를 조합해 만든 가산기
- 전가산기 : 반가산기 2개와 OR 하나를 합쳐 만든 입력 3개, 출력 2개의 가산기
- 인코더 : OR로 구성된 부호기 (n진수를 2진수로)
- 디코더 : AND로 구성된 복호기 (2진수를 n진수로)
- 멀티플렉서 : 여러개의 입력에 오직 한 개를 출력
- 디멀티플렉서 : 한 개의 입력에 여러개 출력

## 순서논리회로 (플립플롭)

- 플립플롭 : 1비트 기억소자이며, 상태를 표시함
- JK 플립플롭 : J와 K가 1일 때 반전, 전상태보수
- RS 플립플롭 : R과 S가 1일 때 허가 금지
- D 플립플롭 : RS의 축소판 (두꺼비집)
- T 플립플롭 : JK의 축소판 (계전기, ON/OFF 스위치)

## 자료의 표현방법

- 10진수 표현방식으론 언팩 방식과 팩 방식이 존재
  - 양수의 경우 C, 음수의 경우 D
  - 10진수 -567은 언팩 방식으로 F5F6D7 (존 10진 형식)
  - 10진수 +1234는 팩 방식으로 0001 234C
- 2진수 표현 방식으론 고정소수점 표현 방식과 부동소수점 표현 방식이 존재
  - 부호와 절대치를 표현하여 1바이트 중 맨 앞 비트에 부호(양수 0, 음수 1)를 기재하고 나머지 7비트를 활용
  - 1의 보수로 표현 시 부호 제외한 나머지 7비트의 모든 값을 각각 NOT
  - 2의 보수로 표현 시 1의 보수로 표현한 후 +1
  - 부동소수점 표현 방식으론 실수를 표현 가능하며, 대단히 크고 작은 수를 표현할 수 있지만 속도가 느림

### 진법 변환

- 10진수 → n진수는 10진수를 n으로 나누어 나머지를 거꾸로 쓰기
- 2진수 → 8진수는 3비트씩 끊어 8진수의 1비트로 만들기
- 2진수 → 16진수는 4비트씩 끊어 16진수의 1비트로 만들기

## 코드와 명령어 형식, 주소 표현 방식

- 명령어는 OP-code와 Operand로 나뉜다.

- 언어번역기 : 사람의 프로그래밍 언어와 기계어를 번역해주는 징검다리
- 종류 : 컴파일러, 인터프리터, 어셈블러, 프리프로세서
- 과정 : 원시프로그램 → 언어번역 → 목적프로그램 → 링커 → 로드모듈 → 로더 → 실행
  - 컴파일러 : 목적프로그램 생성. 속도가 빠름
  - 인터프리터 : 목적프로그램 생성 안함. 대화형 및 줄 단위로 번역

## 명령어(인스트럭션) 형식

- 명렁어는 첫 번째 바이트에 OP-CODE(오피코드, 연산자)와 그 나머지 바이트에 OPERAND(오퍼랜드, 주소)가 존재하는 구성
- 0주소 : 스택 / 1주소 : 누산기 / 2주소 : 보존 안됨 / 3주소 : 보존됨
- 즉시 주소 지정 : 오퍼랜드가 주소가 아닌 실제 데이터
- 직접 주소 지정 : 오퍼랜드의 값이 실제 데이터의 주소
- 간접 주소 지정 : 오퍼랜드 위치의 값이 실제 데이터의 주소 (2회 이상 참조)
- 절대 주소 지정 : 기억장소를 16진수 순서대로 매긴 것
- 인덱스 주소 지정 : 오퍼랜드 값과 인덱스 레지스터 값을 더해 구하는 방식
- 묵시적 주소 지정 : 주소지정 필드가 없고, 0번지 명령어 스택의 TOP 포인터

## 기억장치

- 주기억장치 : RAM(휘발성, 읽기 / 쓰기 가능)
  - DRAM : 충전 필요, 집적도 좋고 전기를 적게 먹음. 가격이 저렴
  - SRAM : 충전 필요 없음. 속도가 빨라 캐시로 쓰임
  - 캐시 메모리 : 중앙처리장치와 주기억장치의 속도 차이 보완 / 자주 사용하는 프로그램 보관
  - ROM : 비휘발성. Read Only Memory의 약자로, 읽기만 가능
  - MASK ROM : 한 번 쓰여진 것을 절대 지우지 못하는 롬
  - PROM : 딱 한 번만 기록 가능
  - EPROM : 자외선으로 재기록 가능
  - EEPROM : 전기적으로 재기록 가능
- 보조기억장치 : 자기디스크(DASD, 동심원(트랙)을 나눈걸 섹터라고 함.), 자기테이프(SASD)
  - 자기 테이프엔 블록화 인수가 존재. IRG와 IBG가 각각 논리 레코드 혹은 물리 레코드 사이의 여백을 의미함

## 입출력 장치

- 입력장치 : OMR, OCR, MICR, BCR, 스캐너, 디지타이저
- 출력장치 : 프린터, 플로터, CRT, LCD, PDP, LED

## 채널

- 채널 : CPU의 제어장치에게 명령을 받아 입출력장치 제어
- 위치 : 주기억장치와 입출력장치의 사이
  - 셀렉터 채널 : 고속의 입출력장치 1개 제어
  - 멀티플렉서 채널 : 저속의 입출력장치 여러개 제어

# 운영체제

## 운영체제의 기능과 성능평가 요소

- 기능 : 사용자로부터 하여금 여러가지 컴퓨터 성능 관리, 편의 제공.
- 종류 : 윈도우, DOS, UNIX, LINUX, OS2, MAC
- 운영체제의 성능평가 요소
  - Throughput(처리능력) : 주어진 시간 내에 얼마나 일을 수행하는가
  - Turn Around Time(응답시간) : 응답시간이 짧을 수록 좋다
  - Realibility(신뢰도) : 정확성
  - Abailability(사용가능도) : 원하는 때에 얼마나 빨리 자원 이용이 가능한지 평가
- 교착상태(DeadLock) : 다중 프로그래밍 상태에서 서로 무한정 지원 대기상태
  - 필요 조건 : 상호배제, 환형대기, 비선점, 점유와 대기
- 스케줄링(작업의 순서 결정) : FIFO, LRU, LFU
- 인터럽트 : 컴퓨터가 갑자기 중단된 상태
  - 위급상황 순서 : 기계고장 → 입출력 → 외부 → SVC → 프로그램 인터럽트
  - 외부 인터럽트 : 오퍼레이터의 콘솔조작 OR 타이머때문에 중단되는 인터럽트
  - 입출력 인터럽트
  - 인터럽트가 걸리게 되면 하던 작업 중지 후 스택에 저장, 인터럽트 해결 후 복귀
- 로드(패치, Fetch) : 하위계층에서 상위계층으로 가는건 로드지만, 상위계층에서 하위계층으로 가는건 스토어와 세이브
- 신뢰성 : 주어진 환경에 관계 없이 처리하는 능력
- 호환성 : 컴퓨터 기종과 관계 없이 프로그램을 수행하는 능력

## 윈도우

- 운영체제 윈도우
  - CUI : 문자 (DOS, UNIX)
  - GUI : 그래픽 (윈도우)
  - 작업표시줄 : 시작, 빠른 실행 영역, 줄, 표시영역. 상하좌우 이동 가능하며 화면 1/2 확대 가능
  - 휴지통 : 삭제파일 임시 보관. 기본적으로 하드디스크 10% 크기, 0 ~ 100% 조절 가능
- 윈도우 98 특징
  - 32비트 OS
  - 플러그 앤 플레이(PNP) : 하드웨어 자동 인식, 설치
  - 공백 포함 255자 (영어 기준)
  - 선점형 멀티테스킹 : 문제시 작업관리자 (Ctrl, Alt, Del)
  - 아이콘, 바로가기 아이콘(원본 영향 X, 여러개가 가능하며 이름은 중복 불가능) 확장자 .LNK
  - Ctrl + ESC 시작 메뉴 표시, Shift + F10 바로가기
- 윈도우 부팅메뉴(F8) : 일반, 안전모드, Command Prompt Only (도스)
- 시작 메뉴 : 프로그램, 문서, 설정(제어판)
- 포맷(기준은 디스켓)
  1. 빠른 포맷 : 이미 포맷하여 쓰던 걸 포맷할 때 사용. 불량섹터 검사 X
  2. 전체 포맷 : 처음 포맷할 때 쓰는 포맷. 불량섹터 검사
  3. 시스템 파일 복사 : 전체파일 삭제 후 시스템파일 복사 (부팅 가능 하드디스크로 만듬)
- 윈도우 탐색기
  - F6, Tab으로 폴더창, 파일창 간 이동 가능
  - 아이콘 정렬 방식에 계단식은 존재하지 않음
  - 클립보드 : 복사 붙여넣기 임시 저장소
  - 창 정렬방식 : 계단식, 가로 / 세로 바둑판식
- 파일 이동 / 복사
  - 같은 드라이브일 경우 기본적으론 이동, Ctrl + 드래그가 복사이며
  - 다른 드라이브일 경우 기본적으론 복사, Shift + 드래그가 이동
- 제어판 (시작 - 설정)
  1. 디스플레이 (바탕화면 → 등록정보)
     - 화면 보호기 : 번인 방지, 암호, 모니터 절전
     - 설정 : 해상도, 색상, 비디오 카드, 모니터 설정
  2. 프로그램 추가 / 제거
     - 시동디스크 작성
  3. 시스템
     - 장치관리자 : 하드웨어들의 목록 보기, 정상작동 여부. 제거 및 드라이버 재설치
  4. 멀티미디어 : 시스템 트레이에 볼륨 조절 표시 / 숨기기
- 인쇄 : 기본 프린터는 1개만. 직접 연결된 프린터는 로컬 프린터, 원격으로 연결된 프린터는 네트워크 프린터
- 보조 프로그램 : 녹음기, 미디어 재생기, 그림판, 메모장
- 시스템 도구
  - 디스크 검사 : 물리 / 논리적 오류를 검사하여 수정
  - 백업, 디스크 공간 늘림, 디스크 정리 (임시파일 삭제하여 용량을 늘림)
  - 디스크 조각모음(Defrag) : 분산파일 모으고 디스크 최적화 (속도 개선, 용량은 그대로)
  - 디스크 등록정보 도구에서 할 수 있는 것 : 디스크 검사, 백업, 조각 모음

# 패키지 활용

## 엑셀

- 스프레드 시트 : 계산, 통계자료, 차트, DB
- 확장자 : .XLS
- 단위 : 셀
- 반복적 작업 : 매크로
- 필요자료 추출 : 필터

## 프레젠테이션(PPT)

- 강연, 발표 목적. 동영상 가능
- 확장자 : .PPT
- 단위 : 쪽, 슬라이드
- 개개의 요소 : 개체
- 흐름 기획 : 시나리오

## 데이터베이스(DB)

- 자료관리 프로그램(액세스, 확장자 MDB)
- 목적 : 중복 최소화, 공유, 보안성 유지, 일관성, 무결성(정확성)
- 장점 : 실시간 접근, 계속 변화, 동시 공유, 내용에 의한 참조
- 단점 : 운영비, 시스템이 복잡, 복구가 어려움
- 기본키 : 필수, 중복불가, NULL값 X
- 나열 : 작은것 부터 큰 순으로 데이터 → 필드 → 레코드 → 테이블
- DBA : 데이터베이스 관리자
- 스키마 : DB의 구조를 논리적으로 정리한 것
  - 종류 : 외부, 내부, 개념
- 폼 : 입출력 그래픽 툴
- 레코드 : 행 / 가로 / 튜플
  - 레코드의 수는 카디널리티, 기수
- 필드 : 열 / 세로 / 어트리뷰트
  - 필드의 수는 디그리, 차수
- 도메인 : 하나의 필드가 취할 수 있는 모든 원자값들의 집합 (속성의 범위)
- DBMS의 필수 기능 : 정의, 조작, 제어 / 개념 → 논리 → 물리

## SQL
- 정의어, 조작어, 제어어 명령어가 존재
- DDL (정의어)
  - CREATE TABLE : 테이블 생성
  - ALTER TABLE : 테이블 수정
  - DROP TABLE : 테이블 삭제
- DML (조작어)
  - INSERT INTO VALUE : 레코드 추가
  - SELECT FROM WHERE : 필드 검색
  - UPDATE SET WHERE : 자료 수정, 갱신
  - DELETE FROM WHERE : 레코드 삭제
  - DISTINCT : 중복되지 않게 검색
  - RESTRICT : 참조중이라면 제거하지 않음
  - CASCADE : 참조중이어도 제거
  - ORDER BY : 정렬
    - ASC : 오름차순
    - DESC : 내림차순
- DCL (제어어)
  - GRANT : 권한 부여
  - REVOKE : 권한 제거
  - COMMIT : 저장, 적용
